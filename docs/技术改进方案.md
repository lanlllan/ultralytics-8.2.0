### 1.2 运单检测任务特点

- **形状不规则**：运单可能折叠、弯曲、部分遮挡，边缘轮廓复杂
- **尺度变化大**：近景占满画面，远景仅占图像一小部分
- **背景干扰**：运单放置在包裹、桌面、传送带等复杂背景上
- **边缘模糊**：胶带覆盖、褶皱、光照不均导致边界不清晰

### 2.1 实验采用的注意力模块

代码中共实现了 5 个注意力类（`conv.py`），但实际实验选取了 **3 种**具有代表性的机制，覆盖不同注意力维度：

| 模块 | 来源 | 注意力维度 | 可学习参数(C=256) | 核心特点 | 实验状态 |
|------|------|----------|----------------|---------|---------|
| **CBAM** | Woo et al., ECCV 2018 | 通道+空间 | ~65,890 | 双维度串联，经典稳定 | **已实验** |
| **SimAM** | Yang et al., ICML 2021 | 3D（通道+空间） | **0** | 完全无参数，基于能量函数 | **已实验** |
| **CoordAtt** | Hou et al., CVPR 2021 | 通道+位置 | ~6,160 | 水平/垂直位置编码，边缘敏感 | **已实验** |
| SEAttention | Hu et al., CVPR 2018 | 通道 | ~8,200 | 挤压激励，FC 瓶颈降维 | 代码备用 |
| ECAAttention | Wang et al., CVPR 2020 | 通道 | ~5 | 1D 卷积替代 FC，极轻量 | 代码备用 |

**选型理由**：
- **CBAM**：通道+空间双维度，经典基线
- **SimAM**：零参数消融对照——若零参数也能提升，说明注意力的"特征选择"本身有效
- **CoordAtt**：位置编码对运单边缘分割尤其有利

实验采用 **2 个插入位置**：Position A（SPPF 后，1 层）和 Position C（Neck 每个 C2f 后，4 层），共 **5 组注意力实验 + 1 组基线对照**。

#### CBAM 实现

```python
# conv.py L313-L324
class CBAM(nn.Module):
    def __init__(self, c1, kernel_size=7):
        super().__init__()
        self.channel_attention = ChannelAttention(c1)
        self.spatial_attention = SpatialAttention(kernel_size)

    def forward(self, x):
        return self.spatial_attention(self.channel_attention(x))
```

公式：$\hat{F} = F \otimes \sigma(\text{Conv}_{1 \times 1}(\text{GAP}(F))) \otimes \sigma(\text{Conv}_{7 \times 7}([\text{Avg}; \text{Max}]))$

#### SimAM 实现（零参数）

```python
# conv.py L362-L375
class SimAM(nn.Module):
    def __init__(self, c1, e_lambda=1e-4):
        super().__init__()
        self.e_lambda = e_lambda

    def forward(self, x):
        b, c, h, w = x.size()
        n = h * w - 1
        mean = x.mean(dim=[2, 3], keepdim=True)
        var = ((x - mean) ** 2).sum(dim=[2, 3], keepdim=True) / n
        y = (x - mean) / (var + self.e_lambda).sqrt()
        return x * y.sigmoid()
```

公式：$\hat{x} = x \odot \sigma\left(\frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}}\right)$

#### CoordAtt 实现

```python
# conv.py L378-L401
class CoordAtt(nn.Module):
    def __init__(self, c1, reduction=32):
        super().__init__()
        mid = max(c1 // reduction, 8)
        self.pool_h = nn.AdaptiveAvgPool2d((None, 1))
        self.pool_w = nn.AdaptiveAvgPool2d((1, None))
        self.conv1 = nn.Conv2d(c1, mid, 1, bias=False)
        self.bn1 = nn.BatchNorm2d(mid)
        self.act = nn.SiLU(inplace=True)
        self.conv_h = nn.Conv2d(mid, c1, 1, bias=False)
        self.conv_w = nn.Conv2d(mid, c1, 1, bias=False)

    def forward(self, x):
        b, c, h, w = x.size()
        x_h = self.pool_h(x)
        x_w = self.pool_w(x).permute(0, 1, 3, 2)
        y = torch.cat([x_h, x_w], dim=2)
        y = self.act(self.bn1(self.conv1(y)))
        x_h, x_w = y.split([h, w], dim=2)
        x_h = self.conv_h(x_h).sigmoid()
        x_w = self.conv_w(x_w.permute(0, 1, 3, 2)).sigmoid()
        return x * x_h * x_w
```

公式：$\hat{x} = x \odot \sigma(W_h \cdot f_h) \odot \sigma(W_w \cdot f_w)$

### 2.7 注意力效果不佳的原因分析

1. **单类别任务过于简单**：分类和定位难度低，"选择性增强"优势无法发挥
2. **nano 模型容量有限**：3.41M 参数，额外注意力在小数据集上引入过拟合风险
3. **mAP@0.5 持平但 mAP@0.5:0.95 下降**：粗略定位不受影响，但精细定位变差——注意力模块在高 IoU 阈值下引入了特征对齐噪声
4. **结论**：注意力机制的实验结果本身也是论文的重要贡献——证明了"并非所有改进都适用于所有场景"

## 三、改进技术二：IoU 损失改进（待实施）

### 3.1 当前实现

**文件**：`ultralytics/utils/metrics.py` L75-L131

`bbox_iou` 函数支持 IoU/GIoU/DIoU/CIoU。在 `BboxLoss.forward`（`loss.py` L76）中固定 `CIoU=True`。

CIoU 公式：

$$\mathcal{L}_{CIoU} = 1 - \text{IoU} + \frac{\rho^2(b, b^{gt})}{c^2} + \alpha v, \quad v = \frac{4}{\pi^2}\left(\arctan\frac{w^{gt}}{h^{gt}} - \arctan\frac{w}{h}\right)^2$$

**CIoU 的不足**：宽高比通过 arctan 比值耦合；对所有样本统一权重；中心距离无方向性。

### 3.2 方案 A1：EIoU（Efficient IoU, 2022）

**论文**：Focal and Efficient IOU Loss for Accurate Bounding Box Regression

$$\mathcal{L}_{EIoU} = 1 - \text{IoU} + \frac{\rho^2(b, b^{gt})}{c^2} + \frac{(w - w^{gt})^2}{c_w^2} + \frac{(h - h^{gt})^2}{c_h^2}$$

**优势**：将宽度差和高度差分开惩罚。对快递单（矩形目标）宽和高可独立优化。不增加参数。

**实现代码**：

```python
if EIoU:
    rho_w2 = (w2 - w1).pow(2)
    rho_h2 = (h2 - h1).pow(2)
    cw2 = cw.pow(2) + eps
    ch2 = ch.pow(2) + eps
    return iou - (rho2 / c2 + rho_w2 / cw2 + rho_h2 / ch2)
```

### 3.3 方案 A2：SIoU（Scylla IoU, 2022）

**论文**：SIoU Loss: More Powerful Learning for Bounding Box Regression

$$\mathcal{L}_{SIoU} = 1 - \text{IoU} + \frac{\Delta_{dist} + \Omega_{shape}}{2}$$

包含角度感知（让预测框沿最短路径趋近 GT）、距离感知、形状感知三个组件。

**优势**：方向感知减少训练震荡，加速收敛。独立约束宽高。

**实现代码**：

```python
if SIoU:
    s_cw = (b2_x1 + b2_x2 - b1_x1 - b1_x2) * 0.5
    s_ch = (b2_y1 + b2_y2 - b1_y1 - b1_y2) * 0.5
    sigma = (s_cw.pow(2) + s_ch.pow(2)).clamp(min=eps).sqrt()
    sin_alpha = s_ch.abs() / sigma
    sin_alpha = sin_alpha.clamp(max=1.0)
    angle_cost = torch.cos(2 * torch.asin(sin_alpha) - math.pi / 2)

    rho_x = (s_cw / cw).pow(2)
    rho_y = (s_ch / ch).pow(2)
    gamma = angle_cost - 2
    dist_cost = 2 - torch.exp(gamma * rho_x) - torch.exp(gamma * rho_y)

    omiga_w = (w1 - w2).abs() / w1.maximum(w2).clamp(min=eps)
    omiga_h = (h1 - h2).abs() / h1.maximum(h2).clamp(min=eps)
    shape_cost = (1 - torch.exp(-omiga_w)).pow(4) + (1 - torch.exp(-omiga_h)).pow(4)

    return iou - 0.5 * (dist_cost + shape_cost)
```

### 3.4 方案 A3：WIoU（Wise IoU, 2023）

**论文**：Wise-IoU: Bounding Box Regression Loss with Dynamic Focusing Mechanism

$$\mathcal{L}_{WIoU} = r_{WIoU} \cdot \mathcal{L}_{IoU}, \quad r_{WIoU} = \exp\left(\frac{(x-x^{gt})^2 + (y-y^{gt})^2}{(W_g^2+H_g^2)^*}\right)$$

v3 版本的非单调聚焦：$r = \frac{\beta^*}{\delta} \cdot r_{WIoU}$，其中 $\beta^*$ 是 IoU Loss 的移动平均。

**优势**：自动降低异常样本（错误标注、边缘模糊）权重。对小数据集特别有效。聚焦中等难度样本。

## 四、改进技术三：分割掩码损失改进（待实施）

### 4.1 当前实现

**文件**：`ultralytics/utils/loss.py` L342-L365

```python
# single_mask_loss() L363-365
pred_mask = torch.einsum("in,nhw->ihw", pred, proto)
loss = F.binary_cross_entropy_with_logits(pred_mask, gt_mask, reduction="none")
return (crop_mask(loss, xyxy).mean(dim=(1, 2)) / area).sum()
```

**BCE 的问题**：对前景/背景不均衡敏感；逐像素损失缺少全局结构感知。

### 4.2 改进方案：添加 Dice Loss

**来源**：V-Net（Milletari et al., 2016）

$$\mathcal{L}_{Dice} = 1 - \frac{2 \sum p_i g_i + \epsilon}{\sum p_i + \sum g_i + \epsilon}$$

**组合策略**：$\mathcal{L}_{mask} = \mathcal{L}_{BCE} + \lambda \cdot \mathcal{L}_{Dice}$（建议 $\lambda = 1.0$）

**优势**：
- 对正负样本不均衡天然鲁棒
- 更关注整体轮廓一致性
- BCE 负责精细边缘，Dice 负责整体形状，两者互补

### 4.3 按需加载设计

**新增配置**：

```python
def single_mask_loss(gt_mask, pred, proto, xyxy, area, use_dice=False, dice_weight=1.0):
    pred_mask = torch.einsum("in,nhw->ihw", pred, proto)

    bce_loss = F.binary_cross_entropy_with_logits(pred_mask, gt_mask, reduction="none")
    loss = (crop_mask(bce_loss, xyxy).mean(dim=(1, 2)) / area).sum()

    if use_dice:
        pred_sigmoid = pred_mask.sigmoid()
        pred_cropped = crop_mask(pred_sigmoid, xyxy)
        gt_cropped = crop_mask(gt_mask, xyxy)
        intersection = (pred_cropped * gt_cropped).sum(dim=(1, 2))
        union = pred_cropped.sum(dim=(1, 2)) + gt_cropped.sum(dim=(1, 2))
        dice = (2 * intersection + 1e-6) / (union + 1e-6)
        dice_loss = ((1 - dice) / area).sum()
        loss = loss + dice_weight * dice_loss

    return loss
```

## 五、改进技术四：标签分配策略调优（待实施）

### 5.1 当前实现

**文件**：`ultralytics/utils/loss.py` L167

```python
self.assigner = TaskAlignedAssigner(topk=10, num_classes=self.nc, alpha=0.5, beta=6.0)
```

对齐度量：$\text{align} = s^\alpha \times u^\beta$

### 5.2 单类别场景的问题

nc=1 时分类分数 $s$ 区分度极低（只有 1 类，很快趋近 1.0），$s^{0.5} \approx 1.0$，导致匹配几乎只看 IoU。topk=10 可能引入过多低质量正样本。

### 5.3 调优建议

```python
self.assigner = TaskAlignedAssigner(topk=5, num_classes=self.nc, alpha=0.3, beta=8.0)
```

- **topk: 10→5**：单类别不需要 10 个候选，减少噪声
- **alpha: 0.5→0.3**：降低低区分度的分类分数影响
- **beta: 6.0→8.0**：增加 IoU 质量权重，提升定位精度
