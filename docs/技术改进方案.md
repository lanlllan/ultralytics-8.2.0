## 一、项目背景

### 1.1 任务描述

| 项目 | 说明 |
|------|------|
| 任务类型 | 实例分割（Instance Segmentation） |
| 检测目标 | 快递运单（单类别，nc=1） |
| 基线模型 | YOLOv8n-seg（3.41M 参数，~12.6 GFLOPs） |
| 数据规模 | 小数据集（约 330 训练图 / 99 验证图） |
| 训练硬件 | NVIDIA RTX 3060 Laptop 6GB |
| 输入分辨率 | 960×960 |
| 下游任务 | 检测 → 透视校正 → OCR 文字提取（对边界精度有容忍） |
| 核心目标 | **提升 Recall（降低漏检率）**+ 维持 mAP@0.5，用于学术论文 |

### 1.2 运单检测任务特点

- **形状不规则**：运单可能折叠、弯曲、部分遮挡，边缘轮廓复杂
- **尺度变化大**：近景占满画面，远景仅占图像一小部分
- **背景干扰**：运单放置在包裹、桌面、传送带等复杂背景上
- **边缘模糊**：胶带覆盖、褶皱、光照不均导致边界不清晰

### 1.4 应用场景分析与指标再评估

#### 1.4.1 OCR 流水线的容错特性

> **参考实现**：`waybill_yolov8_ocr/waybill_ocr/rectifier.py`（角点拟合 + 透视矫正）、`segmentor.py`（YOLO 分割）

本项目的完整流水线为 **检测（掩码）→ 四角点拟合 → 透视矫正 → OCR 文字提取**。角点拟合模块（`rectifier.py`）内置了多层容错机制：

```
掩码 → clean_mask（形态学: 闭运算2次 + 开运算1次 + 最大连通域）
     → convexHull（凸包平滑，消除凹陷/突起）
     → approxPolyDP（7 级 epsilon 逐步放宽: 0.02~0.10 × 周长）
     → [策略 A 失败] minAreaRect 最小外接矩形兜底
     → _expand_quad_with_bbox（掩码 quad 覆盖率 < 75% bbox 时退化为 bbox 四角）
     → order_points → getPerspectiveTransform → warpPerspective
```

**已验证精度**（`rectifier.py` 文档头注明）：
- 规则掩码任意旋转：角点误差 **< 4px**
- **不规则掩码**（突起/缺口/碎片/孔洞）：角点误差 **< 17px**

各类误差对流水线的影响：

| 误差类型 | 下游影响 | 代码容错机制 | 可恢复性 |
|---------|---------|------------|---------|
| **漏检**（FN） | 快递单完全丢失 | 无 | **不可恢复** |
| 误检（FP） | 非运单进入 OCR | OCR 结果可后处理过滤 | 可恢复 |
| 掩码粗糙（IoU ≈ 0.5） | 凸包/minAreaRect 可部分补救，但角点可能偏移较大 | 凸包 + minAreaRect + bbox 兜底 | **部分可补救** |
| 掩码中等（IoU 0.7~0.85） | 凸包后 approxPolyDP 可直接拟合 4 角 | 多级 epsilon + 凸性检查 | **可接受** |
| 掩码高精度（IoU > 0.9） | 对角点无额外收益 | — | 不必要 |

**关键发现**：
1. **漏检是唯一完全不可恢复的错误**——角点拟合代码有兜底，但"没检测到"无法兜底
2. 掩码质量在 IoU ≈ 0.5 时虽有兜底（minAreaRect/bbox），但角点精度已接近极限
3. **IoU ≥ 0.7 时凸包 + approxPolyDP 可直接工作**，是"代码能可靠运作"的实际分界线
4. 部署推理置信度 `conf=0.5`（`config/default.yaml`）比训练评估时的 `conf=0.001` 保守得多，是漏检的直接来源

#### 1.4.2 各 IoU 阈值对角点拟合的实际意义

实际角点拟合链路（`rectifier.py`）为：`clean_mask` → 凸包 → `approxPolyDP`（7 级 epsilon）→ `minAreaRect` 兜底。掩码 IoU 决定了走哪条路径以及最终角点精度：

| IoU 阈值 | 掩码特征 | 走哪条路径 | 角点拟合质量 | COCO 标准 |
|----------|---------|-----------|------------|----------|
| 0.50 | 仅 50% 重叠，凸包后形状仍失真 | minAreaRect 或 bbox 兜底 | 勉强可用但角点偏移大，矫正后变形明显 | mAP@0.5 |
| 0.65 | 大致覆盖正确区域，边缘偏差大 | 凸包 + 高 epsilon 近似 | 角点基本可用，矫正后有轻微变形 | — |
| **0.70** | **形状基本匹配** | **凸包 + approxPolyDP 直接拟合 4 角** | **代码可靠运作的实际分界线** | — |
| **0.75** | **形状良好匹配，边缘偏差适中** | **approxPolyDP 低 epsilon 即可** | **可用** — 角点偏差在可接受范围 | **mAP@0.75**（COCO 标准） |
| 0.85 | 形状高度匹配，边缘偏差小 | 低 epsilon 直接命中 | 良好 — 矫正后文字清晰 | — |
| 0.95 | 像素级精确 | 同上 | 收益微乎其微 | mAP@0.5:0.95 中的一项 |

**核心结论**：
- **IoU ≈ 0.70 是代码容错的下限**——低于此值 `approxPolyDP` 大概率失败，需要退化到 `minAreaRect`/bbox 兜底
- **IoU ≈ 0.75 是"可靠拟合"的分界线**——凸包 + 低 epsilon 即可直接得到有效四角点
- 高于 0.85 边际收益递减，0.95 对角点拟合毫无意义
- 因此 **mAP@0.75**（COCO 标准三指标之一）是本任务最契合实际需求的质量指标

#### 1.4.3 mAP@0.5 与 mAP@0.5:0.95 的各自局限

| 指标 | 局限性 | 对本任务的问题 |
|------|--------|--------------|
| **mAP@0.5** | IoU=0.5 门槛过低 | 通过此阈值的掩码可能严重变形，角点拟合不可靠 |
| **mAP@0.5:0.95** | IoU=0.9~0.95 门槛过高 | 追求像素级精确对角点拟合无意义，白白压低指标 |

已有实验也证实了这一矛盾：所有改进方案在 mAP@0.5 持平的同时 mAP@0.5:0.95 下降，说明瓶颈在 **高阈值（0.9+）的边缘精度**——恰好是对本任务无意义的部分。

#### 1.4.4 优化方向调整

| 原方向 | 新方向 | 理由 |
|--------|--------|------|
| 最大化 mAP@0.5:0.95 | **Recall + mAP@0.75 联合优化** | 漏检不可恢复 + 掩码需满足角点拟合精度 |
| mAP@0.5:0.95 为唯一核心指标 | **mAP@0.75 为掩码质量主指标** | 恰好对应角点拟合的实用门槛 |
| fitness 权重 [0, 0, 0.1, 0.9] | **基于 [R, mAP@0.75, mAP@0.5:0.95] 重新设计** | 去掉 mAP@0.5 和 P，纳入 Recall 和 mAP@0.75 |
| 追求 IoU 0.9+ 的精细边界 | 保证 IoU 0.75+ 前提下提升检出数 | 0.75 以上对角点拟合已足够 |

> **关于自定义指标**：`mAP@0.5:0.9`（`all_ap[:, :9].mean()`，排除 0.95）可作为辅助指标。但论文中应以 COCO 标准三指标（mAP@0.5、mAP@0.75、mAP@0.5:0.95）为主要报告指标，辅以 Recall 和漏检率作为应用导向指标。

### 2.1 实验采用的注意力模块

代码中共实现了 5 个注意力类（`conv.py`），但实际实验选取了 **3 种**具有代表性的机制，覆盖不同注意力维度：

| 模块 | 来源 | 注意力维度 | 可学习参数(C=256) | 核心特点 | 实验状态 |
|------|------|----------|----------------|---------|---------|
| **CBAM** | Woo et al., ECCV 2018 | 通道+空间 | ~65,890 | 双维度串联，经典稳定 | **已实验** |
| **SimAM** | Yang et al., ICML 2021 | 3D（通道+空间） | **0** | 完全无参数，基于能量函数 | **已实验** |
| **CoordAtt** | Hou et al., CVPR 2021 | 通道+位置 | ~6,160 | 水平/垂直位置编码，边缘敏感 | **已实验** |
| SEAttention | Hu et al., CVPR 2018 | 通道 | ~8,200 | 挤压激励，FC 瓶颈降维 | 代码备用 |
| ECAAttention | Wang et al., CVPR 2020 | 通道 | ~5 | 1D 卷积替代 FC，极轻量 | 代码备用 |

**选型理由**：
- **CBAM**：通道+空间双维度，经典基线
- **SimAM**：零参数消融对照——若零参数也能提升，说明注意力的"特征选择"本身有效
- **CoordAtt**：位置编码对运单边缘分割尤其有利

实验采用 **2 个插入位置**：Position A（SPPF 后，1 层）和 Position C（Neck 每个 C2f 后，4 层），共 **5 组注意力实验 + 1 组基线对照**。

#### CBAM 实现

```python
# conv.py L313-L324
class CBAM(nn.Module):
    def __init__(self, c1, kernel_size=7):
        super().__init__()
        self.channel_attention = ChannelAttention(c1)
        self.spatial_attention = SpatialAttention(kernel_size)

    def forward(self, x):
        return self.spatial_attention(self.channel_attention(x))
```

公式：$\hat{F} = F \otimes \sigma(\text{Conv}_{1 \times 1}(\text{GAP}(F))) \otimes \sigma(\text{Conv}_{7 \times 7}([\text{Avg}; \text{Max}]))$

#### SimAM 实现（零参数）

```python
# conv.py L362-L375
class SimAM(nn.Module):
    def __init__(self, c1, e_lambda=1e-4):
        super().__init__()
        self.e_lambda = e_lambda

    def forward(self, x):
        b, c, h, w = x.size()
        n = h * w - 1
        mean = x.mean(dim=[2, 3], keepdim=True)
        var = ((x - mean) ** 2).sum(dim=[2, 3], keepdim=True) / n
        y = (x - mean) / (var + self.e_lambda).sqrt()
        return x * y.sigmoid()
```

公式：$\hat{x} = x \odot \sigma\left(\frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}}\right)$

#### CoordAtt 实现

```python
# conv.py L378-L401
class CoordAtt(nn.Module):
    def __init__(self, c1, reduction=32):
        super().__init__()
        mid = max(c1 // reduction, 8)
        self.pool_h = nn.AdaptiveAvgPool2d((None, 1))
        self.pool_w = nn.AdaptiveAvgPool2d((1, None))
        self.conv1 = nn.Conv2d(c1, mid, 1, bias=False)
        self.bn1 = nn.BatchNorm2d(mid)
        self.act = nn.SiLU(inplace=True)
        self.conv_h = nn.Conv2d(mid, c1, 1, bias=False)
        self.conv_w = nn.Conv2d(mid, c1, 1, bias=False)

    def forward(self, x):
        b, c, h, w = x.size()
        x_h = self.pool_h(x)
        x_w = self.pool_w(x).permute(0, 1, 3, 2)
        y = torch.cat([x_h, x_w], dim=2)
        y = self.act(self.bn1(self.conv1(y)))
        x_h, x_w = y.split([h, w], dim=2)
        x_h = self.conv_h(x_h).sigmoid()
        x_w = self.conv_w(x_w.permute(0, 1, 3, 2)).sigmoid()
        return x * x_h * x_w
```

公式：$\hat{x} = x \odot \sigma(W_h \cdot f_h) \odot \sigma(W_w \cdot f_w)$

### 2.7 注意力效果不佳的原因分析

1. **单类别任务过于简单**：分类和定位难度低，"选择性增强"优势无法发挥
2. **nano 模型容量有限**：3.41M 参数，额外注意力在小数据集上引入过拟合风险
3. **mAP@0.5 持平但 mAP@0.5:0.95 下降**：粗略定位不受影响，但精细定位变差——注意力模块在高 IoU 阈值下引入了特征对齐噪声
4. **结论**：注意力机制的实验结果本身也是论文的重要贡献——证明了"并非所有改进都适用于所有场景"

### 3.1 原始实现

**文件**：`ultralytics/utils/metrics.py` L75-L131

`bbox_iou` 函数支持 IoU/GIoU/DIoU/CIoU。在 `BboxLoss.forward`（`loss.py` L76）中固定 `CIoU=True`。

CIoU 公式：

$$\mathcal{L}_{CIoU} = 1 - \text{IoU} + \frac{\rho^2(b, b^{gt})}{c^2} + \alpha v, \quad v = \frac{4}{\pi^2}\left(\arctan\frac{w^{gt}}{h^{gt}} - \arctan\frac{w}{h}\right)^2$$

**CIoU 的不足**：宽高比通过 arctan 比值耦合；对所有样本统一权重；中心距离无方向性。

### 3.2 方案 A1：EIoU（Efficient IoU, 2022）

**论文**：Focal and Efficient IOU Loss for Accurate Bounding Box Regression

$$\mathcal{L}_{EIoU} = 1 - \text{IoU} + \frac{\rho^2(b, b^{gt})}{c^2} + \frac{(w - w^{gt})^2}{c_w^2} + \frac{(h - h^{gt})^2}{c_h^2}$$

**优势**：将宽度差和高度差分开惩罚。对快递单（矩形目标）宽和高可独立优化。不增加参数。

**实现代码**：

```python
if EIoU:
    rho_w2 = (w2 - w1).pow(2)
    rho_h2 = (h2 - h1).pow(2)
    cw2 = cw.pow(2) + eps
    ch2 = ch.pow(2) + eps
    return iou - (rho2 / c2 + rho_w2 / cw2 + rho_h2 / ch2)
```

### 3.3 方案 A2：SIoU（Scylla IoU, 2022）

**论文**：SIoU Loss: More Powerful Learning for Bounding Box Regression

$$\mathcal{L}_{SIoU} = 1 - \text{IoU} + \frac{\Delta_{dist} + \Omega_{shape}}{2}$$

包含角度感知（让预测框沿最短路径趋近 GT）、距离感知、形状感知三个组件。

**优势**：方向感知减少训练震荡，加速收敛。独立约束宽高。

**实现代码**（含数值稳定性修复）：

```python
if SIoU:
    s_cw = (b2_x1 + b2_x2 - b1_x1 - b1_x2) * 0.5
    s_ch = (b2_y1 + b2_y2 - b1_y1 - b1_y2) * 0.5
    sigma = (s_cw.pow(2) + s_ch.pow(2)).clamp(min=eps).sqrt()
    # 修复: clamp(1-1e-4) + 恒等式替代 asin，消除梯度奇点
    sin_alpha = (s_ch.abs() / sigma).clamp(0, 1.0 - 1e-4)
    angle_cost = 2 * sin_alpha * (1 - sin_alpha.pow(2)).clamp(min=0).sqrt()

    rho_x = (s_cw / cw.clamp(min=eps)).pow(2)
    rho_y = (s_ch / ch.clamp(min=eps)).pow(2)
    gamma = angle_cost - 2
    # 修复: exp 指数截断，防止极端梯度
    dist_cost = 2 - torch.exp((gamma * rho_x).clamp(min=-20))
                   - torch.exp((gamma * rho_y).clamp(min=-20))

    omiga_w = (w1 - w2).abs() / w1.maximum(w2).clamp(min=eps)
    omiga_h = (h1 - h2).abs() / h1.maximum(h2).clamp(min=eps)
    shape_cost = (1 - torch.exp(-omiga_w)).pow(4) + (1 - torch.exp(-omiga_h)).pow(4)

    return iou - 0.5 * (dist_cost + shape_cost)
```

> **数值稳定性修复说明**：原始 SIoU 实现使用 `torch.asin(sin_alpha)`，当 `sin_alpha → 1.0` 时梯度趋于无穷（导数为 $1/\sqrt{1-x^2}$），导致训练在 epoch 6 即出现 loss 发散和 `val/cls_loss = inf`。修复方案：(1) 用数学恒等式 $\cos(2\arcsin x - \pi/2) = 2x\sqrt{1-x^2}$ 完全消除 `asin`；(2) 对 `exp(gamma * rho)` 的指数做 `clamp(min=-20)` 截断。

### 3.4 方案 A3：WIoU（Wise IoU, 2023）

**论文**：Wise-IoU: Bounding Box Regression Loss with Dynamic Focusing Mechanism

$$\mathcal{L}_{WIoU} = r_{WIoU} \cdot \mathcal{L}_{IoU}, \quad r_{WIoU} = \exp\left(\frac{(x-x^{gt})^2 + (y-y^{gt})^2}{(W_g^2+H_g^2)^*}\right)$$

v3 版本的非单调聚焦：$r = \frac{\beta^*}{\delta} \cdot r_{WIoU}$，其中 $\beta^*$ 是 IoU Loss 的移动平均。

**优势**：自动降低异常样本（错误标注、边缘模糊）权重。对小数据集特别有效。聚焦中等难度样本。

**实现代码**（位于 `loss.py` BboxLoss.forward，而非 `metrics.py`，因为 WIoU 修改的是 loss 加权策略）：

```python
if self.iou_type == "WIoU":
    iou = bbox_iou(pred_bboxes[fg_mask], target_bboxes[fg_mask], xywh=False)
    with torch.no_grad():  # wise_scale 全部 detach
        p, t = pred_bboxes[fg_mask], target_bboxes[fg_mask]
        center_dist2 = ((p[:, 0:1] + p[:, 2:3] - t[:, 0:1] - t[:, 2:3]).pow(2)
                        + (p[:, 1:2] + p[:, 3:4] - t[:, 1:2] - t[:, 3:4]).pow(2)) / 4
        cw = p[:, 2:3].maximum(t[:, 2:3]) - p[:, 0:1].minimum(t[:, 0:1])
        ch = p[:, 3:4].maximum(t[:, 3:4]) - p[:, 1:2].minimum(t[:, 1:2])
        c2 = cw.pow(2) + ch.pow(2) + 1e-7
        wise_scale = torch.exp(center_dist2 / c2)
    loss_iou = ((1.0 - iou) * wise_scale * weight).sum() / target_scores_sum
```

> 实现的是 WIoU v1 版本。v3 需要 EMA 维护 loss 移动平均，复杂度更高，v1 已具备动态聚焦的核心能力。

### 3.6 三种 IoU 对比

| 方法 | 年份 | 核心改进 | 对快递单检测的意义 | 实现复杂度 |
|------|------|---------|----------------|-----------|
| **EIoU** | 2022 | 宽高分开惩罚 | 矩形运单的宽高可独立优化 | 低 |
| **SIoU** | 2022 | 角度感知 + 形状感知 | 减少优化震荡，加速收敛 | 中 |
| **WIoU** | 2023 | 动态样本权重 | 降低模糊边缘/错误标注影响 | 中高 |

### 3.7 IoU 损失实验结果

| 实验 | IoU 类型 | mAP@0.5 | mAP@0.5:0.95 | 结论 |
|------|---------|---------|-------------|------|
| baseline | CIoU | 基准 | 基准 | 对照组 |
| eiou | EIoU | ≈ 持平 | ↓ 略低 | 宽高分离惩罚在简单目标上无额外收益 |
| wiou | WIoU | ≈ 持平 | ↓ 略低 | 动态聚焦在小数据集上无显著优势 |

### 3.8 IoU 损失效果分析

IoU 损失改进主要针对**框回归**质量，但在本任务中效果有限：

1. **CIoU 已是近最优**：YOLOv8 默认的 `box=7.5` 等超参数为 CIoU 深度调优，其他 loss 使用相同超参存在天然劣势
2. **单类简单目标**：快递单形状规则、尺度变化有限，框回归难度低，高级 IoU 的优势在困难场景（小目标、密集目标）更明显
3. **分割任务的评测指标（mAP@0.5:0.95）基于掩码 IoU**，而非框 IoU——改进框回归对最终指标的影响是间接的

> 这一结论本身是论文的有价值发现：**IoU 损失改进的有效性与任务复杂度相关，在简单场景中默认 CIoU 已是近最优选择。**

## 四、改进技术三：分割掩码损失改进（已完成实施）

> 与 IoU 损失实验共用同一套脚本：`cmd/yolov8-seg-loss-train.py` / `cmd/yolov8-seg-loss-val.py`

### 4.1 原始实现

**文件**：`ultralytics/utils/loss.py` `single_mask_loss()`

```python
pred_mask = torch.einsum("in,nhw->ihw", pred, proto)
loss = F.binary_cross_entropy_with_logits(pred_mask, gt_mask, reduction="none")
return (crop_mask(loss, xyxy).mean(dim=(1, 2)) / area).sum()
```

**BCE 的三个关键缺陷**：

| 缺陷 | 说明 | 对 mAP@0.5:0.95 的影响 |
|------|------|----------------------|
| 像素独立 | 每个像素单独计算损失，不感知区域整体重叠 | 无法直接优化掩码 IoU |
| 前景稀疏 | 快递单像素占比 < 20%，梯度被背景主导 | 边缘预测偏保守 |
| 边界不敏感 | 边界像素与内部像素权重相同 | 高 IoU 阈值下精度不足 |

### 4.2 改进方案：Dice Loss + BCE 组合

**来源**：V-Net（Milletari et al., 2016）

$$\mathcal{L}_{Dice} = 1 - \frac{2 \sum p_i g_i + \epsilon}{\sum p_i + \sum g_i + \epsilon}$$

**三种配置模式**：

| seg_loss_type | 公式 | 特点 |
|--------------|------|------|
| `BCE` | $L_{BCE}$ | 默认，逐像素交叉熵 |
| `Dice` | $L_{Dice}$ | 纯区域重叠优化 |
| `BCE-Dice` | $L_{BCE} + L_{Dice}$ | **推荐** — 像素级 + 区域级联合 |

**BCE-Dice 互补原理**：
- **BCE** 提供稳定的逐像素梯度信号，保障训练稳定性和边缘细节
- **Dice** 直接优化预测与 GT 的全局重叠比（等价于 soft IoU），与 mAP 评价指标天然对齐
- 两者联合：BCE 管精度，Dice 管覆盖率

### 4.3 实现代码

**Dice Loss 函数**（`loss.py` 顶部新增）：

```python
def dice_loss(pred_mask, gt_mask, smooth=1.0):
    pred = pred_mask.sigmoid()
    pred_flat = pred.flatten(1)
    gt_flat = gt_mask.flatten(1)
    intersection = (pred_flat * gt_flat).sum(1)
    return 1 - ((2.0 * intersection + smooth) / (pred_flat.sum(1) + gt_flat.sum(1) + smooth)).mean()
```

**single_mask_loss 改造**：

```python
@staticmethod
def single_mask_loss(gt_mask, pred, proto, xyxy, area, seg_loss_type="BCE"):
    pred_mask = torch.einsum("in,nhw->ihw", pred, proto)

    if seg_loss_type == "Dice":
        return dice_loss(crop_mask(pred_mask, xyxy), crop_mask(gt_mask, xyxy))

    bce = F.binary_cross_entropy_with_logits(pred_mask, gt_mask, reduction="none")
    bce_loss = (crop_mask(bce, xyxy).mean(dim=(1, 2)) / area).sum()

    if seg_loss_type == "BCE-Dice":
        return bce_loss + dice_loss(crop_mask(pred_mask, xyxy), crop_mask(gt_mask, xyxy))

    return bce_loss  # 默认 BCE
```

## 五、改进技术四：面向召回率的优化策略（待实施）

> **优化动机**：OCR 流水线中漏检不可恢复（详见 §1.4），需要系统性地从训练、评估、推理三个阶段降低漏检率。

### 5.1 当前框架对召回率的不利因素

| 因素 | 当前值 | 对 Recall 的影响 |
|------|--------|-----------------|
| fitness 权重 | `[0, 0, 0.1, 0.9]`（P, R, mAP50, mAP95） | Recall 权重=0，best.pt 选择完全忽略漏检 |
| 推理置信度阈值 | val 时 `conf=0.001`，predict 时 `conf=0.25` | 部署时可能过滤低置信度的正确检测 |
| 标签分配 topk | topk=10 | 单类别场景下可能引入噪声正样本 |
| NMS IoU 阈值 | `iou=0.7` | 对重叠目标过度抑制 |
| 无 TTA | 单尺度推理 | 漏掉特定尺度的困难样本 |

### 5.2 方案 D1：Fitness 函数调整（改变 best.pt 选择标准）

**文件**：`ultralytics/utils/metrics.py` L764-L767

**当前实现**：fitness 基于 `mean_results()` 的 4 个指标加权，完全忽略 Recall 和 mAP@0.75：

```python
def fitness(self):
    w = [0.0, 0.0, 0.1, 0.9]  # [P, R, mAP@0.5, mAP@0.5:0.95]
    return (np.array(self.mean_results()) * w).sum()
```

**问题分析**：
- mAP@0.5 权重仅 0.1：但 IoU=0.5 的掩码对角点拟合不可用，权重应进一步降低
- mAP@0.5:0.95 权重 0.9：被 IoU=0.9~0.95 的极端阈值拉低，且这些阈值对角点拟合无意义
- Recall 权重 0：完全忽略漏检
- **缺失 mAP@0.75**：最契合角点拟合质量门槛的指标未被使用

**建议方案**：绕过 `mean_results()`，直接使用已有属性 `self.mr`、`self.map75`、`self.map`：

```python
def fitness(self):
    """Model fitness: Recall + mAP@0.75 + mAP@0.5:0.95 weighted for OCR pipeline."""
    vals = np.array([self.mr, self.map75, self.map])
    w = np.array([0.2, 0.5, 0.3])  # [Recall, mAP@0.75, mAP@0.5:0.95]
    return (vals * w).sum()
```

> **关键优势**：不修改 `mean_results()` 和 `keys`，日志记录和 CSV 输出完全不受影响。`map75` 属性已存在于代码中（`all_ap[:, 5].mean()`），无需新增。

**权重设计理由**：

| 指标 | 原权重 | 新权重 | 理由 |
|------|--------|--------|------|
| Precision | 0.0 | — | 不参与，误检可被下游过滤 |
| **Recall** | **0.0** | **0.2** | 漏检不可恢复，必须纳入选择标准 |
| mAP@0.5 | 0.1 | — | 不参与，IoU=0.5 掩码对角点拟合不可用 |
| **mAP@0.75** | **—** | **0.5** | **核心**：恰好对应角点拟合的质量门槛 |
| mAP@0.5:0.95 | 0.9 | 0.3 | 保留，反映整体分割质量梯度 |

**为何 mAP@0.75 权重最高（0.5）**：
- IoU=0.75 是掩码形状"基本可用"的分界线——低于此值轮廓锯齿严重，角点拟合不可靠
- 作为 COCO 标准三指标之一，学术可比性好，论文报告自然
- 相比 mAP@0.5:0.95 更聚焦，不被 0.9+ 极端阈值拖累

**实现方式**：建议通过 `default.yaml` 配置化：

```yaml
fitness_metrics: [recall, map75, map]          # 参与 fitness 的指标名
fitness_weights: [0.2, 0.5, 0.3]               # 对应权重
```

> 若不做配置化，直接修改 `metrics.py` 中的 `fitness()` 函数即可（约 3 行改动）。

### 5.3 方案 D2：标签分配策略调优

**文件**：`ultralytics/utils/loss.py` L167

```python
self.assigner = TaskAlignedAssigner(topk=10, num_classes=self.nc, alpha=0.5, beta=6.0)
```

对齐度量：$\text{align} = s^\alpha \times u^\beta$

**单类别场景的问题**：nc=1 时分类分数 $s$ 区分度极低（只有 1 类，很快趋近 1.0），$s^{0.5} \approx 1.0$，导致匹配几乎只看 IoU。

**面向召回率的调优方向**（两组对比实验）：

| 方向 | topk | alpha | beta | 目的 |
|------|------|-------|------|------|
| 基线 | 10 | 0.5 | 6.0 | 默认配置 |
| 精度优先 | 5 | 0.3 | 8.0 | 减少噪声正样本，提升定位精度 |
| **召回优先** | **13** | **0.3** | **4.0** | 增加正样本数量，降低漏检风险 |

**召回优先参数解读**：
- **topk: 10→13**：每个 GT 匹配更多候选 anchor，增加正样本覆盖率
- **alpha: 0.5→0.3**：弱化分类分数（单类别本就无区分度）
- **beta: 6.0→4.0**：降低 IoU 质量门槛，允许更多中等质量匹配

**配置化设计**（`default.yaml`）：
```yaml

tal_alpha: 0.5   # 默认
tal_beta: 6.0    # 默认
```

### 5.4 方案 D3：推理阶段置信度阈值优化

推理时 `conf` 阈值直接决定漏检率——阈值越低，漏检越少，但误检增加。

**当前部署状态**（`waybill_yolov8_ocr/config/default.yaml`）：

```yaml
yolo:
  conf_threshold: 0.5   # ← 当前部署值，相当保守
```

`conf=0.5` 意味着模型置信度低于 50% 的检测直接丢弃。考虑到小数据集（330 训练图）训练的模型，部分真实快递单的预测置信度可能在 0.3~0.5 区间，被直接过滤掉——**这是当前漏检的直接来源**。

**阈值选择矩阵**（需实验验证）：

| conf 阈值 | 预期效果 | 适用场景 | 误检处理 |
|-----------|---------|---------|---------|
| **0.50（当前部署）** | 高精度、低召回 | 保守部署 | 几乎无误检 |
| 0.30 | 平衡精度与召回 | 通用检测 | 轻微误检 |
| **0.20（建议起点）** | 降低漏检，误检可控 | OCR 流水线 | OCR 结果可过滤 |
| 0.10 | 显著降低漏检 | 高召回要求 | 需后处理过滤 |

> **误检的低成本**：误检进入 OCR 后，若 OCR 未识别到有效文字（如地址、电话、姓名等模式），可被后处理逻辑自然过滤。因此降低 conf 阈值的代价远低于"漏检一张快递单"。

**实验方法**：在验证集上扫描 conf 阈值，绘制 Recall-Precision 曲线和 Miss Rate-FPPI 曲线，选择最优工作点。

```python
for conf in [0.05, 0.10, 0.15, 0.20, 0.25, 0.30, 0.40, 0.50]:
    metrics = model.val(data=DATA_YAML, conf=conf, imgsz=960)
    print(f"conf={conf:.2f}  Recall={metrics.seg.mr:.4f}  Precision={metrics.seg.mp:.4f}")
```

### 5.5 方案 D4：测试时增强（TTA）

YOLOv8 内置 TTA 支持（多尺度 + 翻转），对漏检率有直接帮助：

```bash
# 启用 TTA 推理
yolo predict model=best.pt source=images/ augment=True
```

TTA 通过在不同尺度和翻转条件下推理并合并结果，可以找回单尺度推理中遗漏的目标。代价是推理速度降低 2~3 倍。

### 5.6 方案 D5：自定义评估指标

在 COCO 标准指标（mAP@0.5、**mAP@0.75**、mAP@0.5:0.95）基础上新增面向应用的指标：

| 指标 | 公式/实现 | 意义 | 代码现状 |
|------|----------|------|---------|
| **mAP@0.75** | `all_ap[:, 5].mean()` | 角点拟合质量门槛，COCO 标准 | **已有**（`self.map75`） |
| **漏检率** | $1 - \text{Recall}$ | 最核心的应用指标 | 需额外输出 |
| **F2 Score** | $F_2 = \frac{5 \cdot P \cdot R}{4P + R}$ | 对 Recall 加权的 F-score | 需新增 |
| **mAP@0.5:0.9** | `all_ap[:, :9].mean()` | 去掉 IoU=0.95 的宽松版 mAP | 需新增 |
| **Recall@conf=0.15** | 指定 conf 阈值的 Recall | 部署工作点的漏检率 | 需配合阈值扫描 |

**新增属性实现**（`metrics.py` Metric 类）：

```python
@property
def map90(self):
    """mAP@0.5:0.9 — excludes IoU=0.95 threshold."""
    return self.all_ap[:, :9].mean() if len(self.all_ap) else 0.0

@property
def f2(self):
    """F2 score — weights Recall higher than Precision."""
    p, r = self.mp, self.mr
    return (5 * p * r) / (4 * p + r + 1e-16)
```
